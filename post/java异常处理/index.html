错误类型

  1. 用户的输入错误
    除了不可避免的键盘输入错误，有些用户不清楚格式要求甚至不遵守格式要求。例如用户请求一个格式错误的URL，程序如果不检查，网络层就会报警。
  2. 设备错误
    硬件有发生故障的可能。例如网页可能临时性不能浏览；打印机在打印过程中没纸了。
  3. 物理限制
    磁盘已满，没有可用的存储空间。
  4. 程序错误
    程序有可能无法正确执行。例如返回一个错误的答案；调用了错误的方法。

错误处理

  1. 向用户通报错误，并返回到一种安全状态；
  2. 保存用户的操作结果；
  3. 允许用户以妥善的形式退出程序；

错误处理方法包括设置全局变量、返回错误码和异常处理；在Java中，如果某个方法不能按照正常的路径地完成任务，就可以通过另外的路径退出该方法，在这种情况下，方法不返回任何值，而是抛出一个封装了错误信息的对象。需要注意的是，这个方法将会立即退出，并不返回任何值。此外，调用这个方法的代码也将无法继续执行，而是，异常处理机制开始搜索能够处理这种异常状况的异常处理器。

java异常分类

java中所有异常都是由Throwable类派生而来，分为两个分支：Error和Exception。

Error

Error类层次结构描述java运行时系统的内部错误和资源耗尽错误。程序不应该抛出这种类型的对象。如果出现这样的内部错误，除了通告给用户，并尽力使程序安全终止之外，再也无能为力了。

Exception

Exception层次结构分解为两个分支：RuntimeException和其他异常。由程序错误导致的异常属于RuntimeException，程序本身没有问题，但由于像I/O错误这类问题导致的异常属于其他异常。

RuntimeException派生出的异常：

错误的类型转换
数组访问越界
访问空指针

不是RuntimeException派生出的异常：

试图在文件尾部后面读取数据
试图打开一个不存在的文件
试图根据给定的字符串查找Class对象，而这个字符串表示的类并不存在

五个关键字

java异常处理主要由5个关键字控制，分别为：try、catch、throw、throws和finally。

try: 一个代码防护块，检测方法体中是否会发生异常。
catch: 捕获try代码块中发生的异常，并进行处理，使程序不受影响而继续执行下去。
throw: 在程序中明确引发异常。
throws: 方法引发异常，一个或者多个，但方法自身不处理异常，而是将异常抛给调用者。
finally: 不管异常是否发生，都会被执行的代码。
